Navegando el sistema de archivos de ROS.

Paquetes: ROS se compone de archivos en disco, llamados paquetes, que son la unidad básica de organización 
Manifests (package.xml): Es una descripción de un paquete, sirve para definir dependencias entre los paquetes y capturar metadatos sobre el paquete, como versión, autor etc.

$rospack permite obtener información acerca de los paquetes 
e.g. $ rospack find roscpp 
regresa /opt/ros/kinetic/share/roscpp
que es donde está instalado rospp

$ roscd te cambia directamente a un paquete o stack
e.g. $roscd roscpp
	 $pwd /opt/ros/kinetic/share/roscpp

$roscd log Te dirige al directorio donde ROS guarda logs de programas ejecutados
e.g. $roscd log 
	 $pwd
	/home/leguiart/.ros/log/b59598b8-ac09-11e8-9bfe-3ca067eddb03

$rosls como roscd, no necesita de rutas absolutas, basta con darle un nombre de paquete 
e.g. $rosls eguiarte_morett
CMakeLists.txt  include  package.xml  prueba.txt  src

Funcion de autocompletado: tab una vez para autocompletar un cierto nombre
e.g. $rosls egui <<TAB>>
	 $rosls eguiarte_morett

Tab dos veces enlista todas las coincidencias de paquetes

Understanding rosnodes

Nodes: A node is an executable that uses ROS to communicate with other nodes.

Messages: ROS data type used when subscribing or publishing to a topic.

Topics: Nodes can publish messages to a topic as well as subscribe to a topic to receive messages.

Master: Name service for ROS (i.e. helps nodes find each other)

rosout: ROS equivalent of stdout/stderr

roscore: Master + rosout + parameter server (parameter server will be introduced later)

Enlistar los nodos activos:
$ rosnode list

Informacion sobre un nodo especifico:
$ rosnode info /rosout
